Mindset First

A Reliable Pandas Workflow

Before writing code, define a repeatable flow. This prevents random trial-and-error and helps you present clearly in interviews.

The 7-Step Mental Model

1. Load and profile: shape, dtypes, sample rows, null summary, duplicates.
2. Standardize schema: column names, whitespace, casing, known category labels.
3. Fix data types: parse dates, convert numeric text, set category types.
4. Handle missing and invalids: drop or impute based on business logic.
5. Handle outliers and inconsistencies: clip, winsorize, or flag.
6. Transform and aggregate: feature engineering, grouping, pivots.
7. Validate and export: assertions, sanity checks, save cleaned artifact.

Data Quality

Missing Values and Type Fixing

Most real datasets fail here first. Strong cleaning skills separate good candidates from great ones.

Handle Missing Values Intentionally

Column TypeTypical StrategyExample
NumericMedian imputationdf["monthly_spend"] = df["monthly_spend"].fillna(df["monthly_spend"].median())
CategoricalMode or "unknown"df["plan"] = df["plan"].fillna("unknown")
DateDomain-based fill or keep missingDo not blindly forward-fill

Convert Messy Strings to Numeric and Date

# Remove currency symbols and commas, then convert df["monthly_spend"] = ( df["monthly_spend"] .astype(str) .str.replace(r"[^0-9.-]", "", regex=True) .replace("", pd.NA) .pipe(pd.to_numeric, errors="coerce") ) # Convert text counts to numeric df["tickets_last_30d"] = pd.to_numeric(df["tickets_last_30d"], errors="coerce") # Parse mixed date formats df["signup_date"] = pd.to_datetime(df["signup_date"], errors="coerce", utc=True)
Keep errors="coerce" during first pass. It reveals bad values as NaN instead of crashing, so you can inspect and fix systematically.

Custom Cleaning Helper Functions

def normalize_text(series: pd.Series) -> pd.Series: return ( series.astype("string") .str.strip() .str.lower() .str.replace(r"\s+", " ", regex=True) ) def cap_outliers(series: pd.Series, low=0.01, high=0.99) -> pd.Series: lo, hi = series.quantile([low, high]) return series.clip(lo, hi) df["region"] = normalize_text(df["region"]) df["monthly_spend"] = cap_outliers(df["monthly_spend"])
What to Look For

Data Cleaning Checklist

Use this order in your project and in interviews when explaining your approach.

Order of Operations

Schema checks: required columns, primary key uniqueness, row count baseline.
Text normalization: trim, lowercase, unify labels and typos.
Type conversion: numeric/date conversion with coercion and diagnostics.
Missing value strategy: per-column treatment, not one-size-fits-all.
Duplicate handling: exact row duplicates and duplicate IDs with tie-breakers.
Outlier policy: remove, cap, or keep and flag.
Feature creation: make analysis-friendly columns after cleaning.
Validation: assertions for impossible values and null rates.

Validation Assertions

assert df["customer_id"].isna().sum() == 0 assert df["customer_id"].nunique() == len(df) assert (df["monthly_spend"] >= 0).all() assert set(df["churned"].dropna().unique()).issubset({0, 1}) assert df["signup_date"].isna().mean() < 0.05

These checks help you catch accidental corruption before modeling.

Step-by-Step Cleaning Code

import pandas as pd import numpy as np # 1) Load safely and preserve raw raw = pd.read_csv("messy_customers.csv") df = raw.copy() # 2) Standardize column names (df.columns) df.columns = ( df.columns .str.strip() .str.lower() .str.replace(r"[^a-z0-9]+", "_", regex=True) .str.strip("_") ) # 3) Normalize text fields text_cols = ["customer_id", "region", "plan", "notes", "churned"] for c in text_cols: df[c] = df[c].astype("string").str.strip() df["region"] = ( df["region"] .str.lower() .replace({"n/a": pd.NA, "na": pd.NA, "": pd.NA}) ) df["plan"] = df["plan"].str.lower().replace({"": pd.NA}) # 4) Parse numerics (remove currency symbols, spaces, commas) for c in ["monthly_spend", "tickets_last_30d"]: df[c] = ( df[c].astype("string") .str.replace(r"[^0-9.-]", "", regex=True) .replace("", pd.NA) .pipe(pd.to_numeric, errors="coerce") ) # 5) Parse dates (mixed formats) df["signup_date"] = pd.to_datetime(df["signup_date"], errors="coerce", utc=True) # 6) Convert churned to binary churn_map = {"0": 0, "1": 1, "no": 0, "yes": 1, "false": 0, "true": 1} df["churned"] = df["churned"].str.lower().map(churn_map) # 7) Remove exact duplicates df = df.drop_duplicates() # 8) Basic validity rules df.loc[df["monthly_spend"] < 0, "monthly_spend"] = pd.NA # 9) Impute missing values (example policy) df["monthly_spend"] = df["monthly_spend"].fillna(df["monthly_spend"].median()) df["tickets_last_30d"] = df["tickets_last_30d"].fillna(0) df["region"] = df["region"].fillna("unknown") df["plan"] = df["plan"].fillna("unknown") # 10) Outlier treatment: cap monthly_spend at 99th percentile cap_hi = df["monthly_spend"].quantile(0.99) df["monthly_spend"] = df["monthly_spend"].clip(upper=cap_hi) # 11) Feature engineering df["signup_day"] = df["signup_date"].dt.date df["spend_per_ticket"] = np.where( df["tickets_last_30d"] > 0, df["monthly_spend"] / df["tickets_last_30d"], np.nan ) # 12) Final type optimization for c in ["region", "plan"]: df[c] = df[c].astype("category") df["tickets_last_30d"] = df["tickets_last_30d"].astype("int64") # 13) Validation checks assert df["customer_id"].isna().sum() == 0 assert (df["monthly_spend"] >= 0).all() assert set(df["churned"].dropna().astype(int).unique()).issubset({0, 1}) # 14) Save cleaned data df.to_csv("customers_cleaned.csv", index=False)

Why This Flow Works

  • It keeps original data intact and uses deterministic steps.
  • It separates parsing, validity rules, imputation, and feature engineering.
  • It includes explicit assertions so errors are caught early.
  • It ends with a clean artifact ready for EDA or modeling.
