🎲 Probability Distributions — The Big Five
Know the formula · the shape · the use case

Every distribution answers a specific type of question. Match the question to the distribution — that's 80% of the battle.

📌 Binomial Distribution
Discrete · Counting successes

"How many successes in n independent trials?"

Think: coin flips, ad clicks, user signups — anything where each trial is binary (success/fail).

P(X = k) = C(n,k) · p^k · (1−p)^(n−k)
Mean: μ = np
Variance: σ² = np(1−p)
Ad click-through rates User signups A/B testing outcomes
P(X=k) for n=20, p=0.4
🎯 Interview Tip
When n is large, the Binomial approximates a Normal distribution (CLT!). An interviewer might ask you to justify this switch — say: "As n → ∞, by CLT, the binomial converges to N(np, np(1−p))."
📌 Poisson Distribution
Discrete · Events in an interval

"How many events in a fixed time/space interval?"

Think: support tickets per day, website visits per hour, fraud events per week.

P(X = k) = (e^−λ · λ^k) / k!
Mean: μ = λ
Variance: σ² = λ
🔑 Key Insight: Mean = Variance. If you see data where mean ≈ variance, think Poisson immediately.
Support ticket volume Fraud events/week Website traffic
P(X=k) for λ=5
🎯 Interview Tip
Poisson is the discrete event counter. Exponential (below) is its continuous-time twin — Poisson counts events, Exponential measures time between events.
📌 Exponential Distribution
Continuous · Time between events

"How long until the next event?"

If Poisson counts events, Exponential times the gaps between them.

f(x) = λe^(−λx) for x ≥ 0
Mean: 1/λ
Variance: 1/λ²
🔑 Memorylessness: P(X > s+t | X > s) = P(X > t). If you've waited 5 min, remaining wait is same as if you just arrived.
Customer wait times Time until churn Time between transactions
f(x) for λ=1 (blue), λ=0.5 (green)
🎯 Interview Tip
Memorylessness is a very common interview topic. Example: "A server has been running for 2 hours with no failure. Given the exponential failure model, what's the expected additional runtime?" Answer: same as the original expected lifetime (1/λ).

🌊 Central Limit Theorem (CLT)
The Most Important Theorem in Statistics

The CLT is the reason almost all of classical statistics works. It bridges the gap between messy real-world data and the elegant mathematics of the Normal distribution.

If X₁, X₂, ... Xₙ are i.i.d. with mean μ and variance σ²,
then: X̄ ~ N(μ, σ²/n) as n → ∞
The Intuition

Take any distribution — even a weird, skewed, lopsided one. Start drawing samples of size n. Calculate the mean of each sample. As n grows, those sample means form a perfect bell curve. Every time.

Why the CLT Matters for Interviews
1
Justifies Z-tests and t-tests on any data

Even if your raw data isn't normal (e.g., skewed revenue data), you can still use normal-based tests on the sample mean if n ≥ 30.

2
Foundation of all A/B testing

When comparing conversion rates, the CLT is what lets you use a Z-test even though individual outcomes are Bernoulli (0 or 1).

3
Connects Binomial → Normal

For large n, Binomial(n,p) ≈ N(np, np(1−p)). Interviewers ask this often.

🎯 Uber Interview Question (from your book)
"Explain the Central Limit Theorem. Why is it useful?"

Model Answer: The CLT states that the distribution of sample means approaches a Normal distribution as sample size grows, regardless of the population's original distribution. It's useful because it lets us apply parametric statistical tests (like Z-tests) to real-world data that isn't normally distributed, as long as our sample size is large enough (typically n ≥ 30).
🔔 The Normal Distribution — Deep Dive
The bell curve · Symmetric · Described entirely by μ and σ

When someone says a variable is "normally distributed," they mean it follows this shape: symmetric around the mean, with most values clustered near the center and progressively fewer toward the extremes.

🎯 Interview Application
If test scores are N(70, 100) — mean 70, std 10 — then 95% of students score between 50 and 90. A score of 95 is 2.5σ above mean: extremely unusual. This logic is core to anomaly detection.
Standard Normal Z-Score

Any Normal distribution can be converted to Standard Normal N(0,1) using:

Z = (X − μ) / σ

This tells you: "how many standard deviations is this value from the mean?"

Example:

Heights ~ N(170cm, 100). What's the Z-score for someone 185cm tall?

Z = (185 − 170) / 10 = 1.5

They're 1.5 standard deviations above average. Roughly top 7%.

Why Z-scores matter:

They standardize values across different scales, making them comparable. Critical for feature scaling in ML, outlier detection, and hypothesis testing.

🔬 Hypothesis Testing
The Statistical Engine Behind A/B Testing

Hypothesis testing is a framework for making decisions from data. Every A/B test you'll run as a data scientist is built on this foundation.

The 3-Step Framework
1
State your Hypotheses

H₀ (Null): Baseline — "no effect," "no difference," "nothing changed"
H₁ (Alternative): What you're trying to prove

2
Calculate test statistic & p-value

The p-value = probability of observing your result assuming H₀ is true. Smaller p-value = stronger evidence against H₀.

3
Compare p-value to α (significance level)

Standard: α = 0.05. If p-value < α → reject H₀. If p-value ≥ α → fail to reject H₀.

One-Tailed vs Two-Tailed Tests
Two-Tailed Test
H₀: μ = μ₀ vs H₁: μ ≠ μ₀

Use when you care about any difference (either direction). Most common in A/B tests.

e.g., "Does the new UI change engagement at all?"

One-Tailed Test
H₀: μ = μ₀ vs H₁: μ > μ₀

Use when you only care about one direction.

e.g., "Does the new feature increase conversion?"

Rejection regions for two-tailed test (α=0.05, shaded red)
Real Example: Uber Eats Email Campaign
1
State hypotheses

H₀: email group conversion = control group conversion
H₁: email group conversion > control group conversion

2
Run Z-test (large sample, comparing proportions)

Compute Z = (p̂₁ − p̂₂) / SE. Get p-value from normal table.

3
Decision

If p-value < 0.05 → ✅ Email campaigns significantly boost conversion. Roll it out!

🎯 P-Value Intuition
"The p-value is NOT the probability that H₀ is true. It's the probability of seeing results as extreme as yours, assuming H₀ is true." — Interviewers specifically test this misconception.
⚠️ Type I & Type II Errors
False Positives · False Negatives · The Power Trade-off

Interviewers love these — especially asking you to reason about which error is worse in a given scenario.

🚨 Type I Error (α)

False Positive — You detected an effect that doesn't exist.

α = P(Reject H₀ | H₀ is true)

Standard level: α = 0.05 (5% chance of false positive)

Example: You roll out a new feature thinking it helped engagement. It didn't — you got fooled by random variation.
⚠️ Type II Error (β)

False Negative — You missed a real effect.

β = P(Fail to reject H₀ | H₁ is true)

Power = 1 − β (want this high, ~0.8+)

Example: A drug actually works, but your study didn't detect the effect — and you abandoned a cure.
The Trade-off — and How to Think About It

You cannot minimize both errors simultaneously. Making α smaller (fewer false positives) means more false negatives, and vice versa. You choose based on the cost of each error.

As α decreases, β increases (power decreases)
🎯 Classic Interview Scenario
"In a cancer screening test, which error is worse?"

→ Type II (False Negative) — missing real cancer is catastrophic. So we lower the detection threshold, accepting more false positives (unnecessary follow-up tests) to catch more true cases.

"In a spam filter, which is worse?"
→ Type I (False Positive) — flagging real emails as spam causes users to miss important messages. So we're more conservative and accept more spam getting through.
⚖️ Z-test vs t-test vs Chi-Squared
Choosing the right test is half the answer

Interviewers expect you to instantly know which test fits the situation. Here's the decision logic.

Quick Decision Guide
What type of data?
├─ Continuous (means)
│   ├─ n ≥ 30 or σ² known? → Z-test
│   └─ n < 30 and σ² unknown? → t-test
└─ Categorical
     └─ Comparing counts/independence? → Chi-squared

Memory hook: "Z when you Know, t when you Don't, χ² for Categories"

Z-test Deep Dive
Z = (X̄ − μ₀) / (σ / √n) ~ N(0,1)

Use when:

  • Sample size n ≥ 30 (CLT kicks in)
  • Population variance σ² is known
  • Comparing proportions (A/B testing)
Z-distribution with α=0.05 critical values
t-test Deep Dive
t = (X̄ − μ₀) / (s / √n) ~ t(n−1 df)

Use when:

  • Small sample size (n < 30)
  • Population variance σ² is unknown (use sample variance s²)
  • Degrees of freedom = n − 1

The t-distribution has heavier tails than Normal — it's more conservative with small samples, which is appropriate since there's more uncertainty.

t-distribution vs Normal (fatter tails)
🎯 Facebook Interview Question
"What is a Z-test and when would you use it versus a t-test?"

Both test means, but the Z-test assumes known variance and uses the Normal distribution, while the t-test estimates variance from the sample and uses the t-distribution — which has heavier tails to account for the additional uncertainty in small samples. As n → ∞, the t-distribution converges to the Normal.
Chi-Squared Test
χ² = Σ (Observed − Expected)² / Expected

Two use cases:

1. Goodness of Fit

Does observed data match expected distribution?

e.g., Is a die fair? Roll 600 times, expect 100 of each face. How far is reality from expectation?

2. Test of Independence

Are two categorical variables related?

e.g., Is gender independent of product preference? Build a contingency table and run chi-squared.
